### HTTP와 HTTPS의 차이

- HTTP(HyperText Transfer Protocol)는 클라이언트와 서버 간의 데이터를 텍스트 기반으로 주고받는 프로토콜
- HTTP는 보안 기능이 없어 중간자 공격, 데이터 도청 등의 위험이 존재, 이를 보완한 것이 HTTPS
- HTTPS는 SSL/TLS 프로토콜을 기반으로 동작하며 데이터 암호화, 무결성 보장, 서버 인증을 제공
- 클라이언트는 서버의 인증서를 확인하고 공개키-개인키 기반의 암호화를 통해 안전하게 통신
- 포트 번호 차이 (HTTP-80, HTTPS-443)

### ETC. HTTP 버전별 특징

- HTTP 1.0
    - 요청마다 새 연결 생성 - 헤더 캐싱 부족, 연결 재사용 불가
- HTTP 1.1
    - Persistent 연결 (Keep-Alive)
    - 하지만 단일 TCP 연결을 사용할 때, HOL Blocking 문제가 발생
    - 이를 다중 TCP 연결로 해결하려고 했지만, 서버 리소스를 많이 소비하고 bandwidth도 낭비
- HTTP 2.0
    - 단일 TCP 연결에서 다중 요청을 처리해서 TCP 연결도 줄이고, HOL blocking도 제거
    - 멀티플렉싱 = 프레임 + 스트림 구조, 하나의 TCP 연결 위에서 여러 스트림이 동작하고, 각각의 스트림은 프레임 단위로 쪼개져서 전송
    - 2개의 객체가 있다고 할 때, 더 큰 객체의 첫 번째 프레임을 전송하고 다음 전송에는 더 작은 객체의 첫 번째 프레임을 전송하는 방식이 Framing 방식
- HTTP 3.0
    - HTTP 2.0은 애플리케이션 계층에서 멀티플렉싱을 구현했지만, 여전히 TCP 위에서 동작하기 때문에, 전송 계층에서 패킷 손실이 발생하면 다른 스트림의 데이터까지 지연되는 HOL Blocking이 발생
    - UDP 기반의 QUIC은 스트림 단위로 독립적인 패킷 처리, 따라서 전송 계층에서의 HOL Blocking을 완전 제거
